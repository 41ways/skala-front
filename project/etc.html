과제 요구사항 정리

각 과제 요구사항을 어느 파일 · 몇 번째 줄 · 어떤 코드로 구현했는지 정리했습니다.
GitHub 저장소에 올라간 파일 기준이며, 필수 요소를 그대로 충족한 부분과 가독성·사용성을 위해 추가로 발전시킨 부분을 함께 표기했습니다.

충족 요구 요소를 그대로 구현 발전 요구를 넘어 확장·개선 참고 형태가 달라진 부분

1. Web 개요 index.html

01

index.html 생성 · 타이틀/본문

발전

index.html · 8줄(title), 24~29줄(인트로 타이틀)

<title>Hello, world! I'm 한결</title>
<p class="intro__name" id="introName">I'm 한결</p>

최초 과제는 title을 "Welcome SKALA"로 만드는 것이었는데, 이후 포털 Hub 과제로 페이지를 새로 디자인하면서 인트로 화면으로 바꿨습니다. "Hello, world!"를 setTimeout으로 한 글자씩 타이핑하고, 글리치(지지직) 효과 뒤 "I'm 한결"으로 전환되게 만들었습니다.
참고: 원래 h1 "환영 인사"와 p 문구는 이 과정에서 빠졌습니다.

＋ 추가로 신경 쓴 부분 가독성 · 사용성

· 코드 가독성: 코드마다 @동작 · @주의 · @확장 · @필수 같은 주석 태그를 달아, 이 코드가 무엇을 하고 과제 필수 요소가 어디인지 바로 보이게 했습니다. CSS 클래스는 BEM(block__element--modifier) 규칙으로 지어 구조를 명확히 했습니다.

· 사용성: 미니게임·날씨·가방에 열기/닫기(토글) 버튼과 결과 영역을 둬서 다루기 쉽게 했고, 모든 페이지의 메뉴바·헤더·오버레이 메뉴를 똑같이 통일했습니다.

· 공통 UX·접근성: 어느 페이지에서든 우하단 미니 LP 플레이어로 음악을 이어 재생/일시정지할 수 있고, 영상 페이지는 가로 스크롤바로 필름을 넘겨봅니다. 키보드 사용자를 위한 본문 바로가기(skip-link), 스크린리더용 설명(aria-label), 움직임 최소화 설정 대응(prefers-reduced-motion)까지 넣어 접근성도 챙겼습니다.

· 미디어 정리: media/ 폴더를 페이지별(home·profile·album·photos·videos·music·holiday)로 정리하고, 파일 추가 규칙을 media/README.md에 적어놨습니다.
